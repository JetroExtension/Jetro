# Thin Prompt Reference

This document explains what the thin prompt (CLAUDE.md) is, how it works, and how to customize it.

## What is the Thin Prompt?

The thin prompt is a markdown file (`CLAUDE.md`) that Jetro auto-generates in every workspace on activation. AI agents read this file to understand what tools are available and how to use them.

It's called "thin" because it contains only routing information — tool signatures, skill/template names, and basic usage rules. It does NOT contain your system prompt or proprietary methodology (those are delivered separately via `wrapResponse()`).

## Where is it generated?

The thin prompt is generated by `bootstrapService.ts → injectAgentContext()`. It writes to:

- `CLAUDE.md` — VS Code / Claude Code
- `.cursor/rules/agent.mdc` — Cursor
- `.agents/rules/agent.md` — Antigravity
- `.github/copilot-instructions.md` — GitHub Copilot
- `.windsurfrules` — Windsurf
- `AGENT.md` — Generic fallback

All files contain the same content, formatted for each editor's convention.

## Default Content

The default thin prompt includes:

1. **Canvas System** — element types, frame rules, jet:refresh event format
2. **C2 Mode** — inter-frame wiring with `__JET` SDK
3. **Tools table** — all MCP tools with one-line descriptions
4. **Python SDK** — `jet.api`, `jet.market`, `jet.connectors` usage
5. **Quick troubleshooting** — 3 common fixes
6. **Available skills** — auto-listed from `agent/skills/*.json`
7. **Available templates** — auto-listed from `agent/templates/*.json`
8. **Workspace layout** — directory structure

## How to Customize

Edit the `content` array in `extension/src/services/bootstrapService.ts → injectAgentContext()`.

### Adding Behavioral Doctrine

Add a "Working Style" section to guide how the agent interacts:

```typescript
// Add after the "---" separator, before "## Canvas System"
"## Working Style",
"",
"When a user gives you a complex task, discuss the approach before building.",
"Ask clarifying questions. Suggest alternatives. Get confirmation before executing.",
"For simple requests, just do it.",
```

### Adding Domain-Specific Instructions

```typescript
"## Domain Guidelines",
"",
"- When analyzing data, always check for outliers and missing values",
"- Prefer interactive charts over static images",
"- Always include data sources and timestamps in reports",
```

### Adding Language Instructions

```typescript
"## Language",
"",
"Respond in whatever language the user writes in.",
"Code stays in English, but comments and explanations adapt to the user's language.",
```

## MCP Tool Definitions

Tools are defined in `mcp-server/src/index.ts` in the `TOOLS` array. Each tool has:

```typescript
{
  name: "jet_render",           // Tool name (agent calls this)
  description: "...",           // Shown to agent — keep under 1500 chars
  inputSchema: {                // JSON Schema for parameters
    type: "object",
    properties: { ... },
    required: ["type", "data"]
  }
}
```

The thin prompt lists tools in a table for quick reference. The full tool descriptions and schemas are served via MCP to the agent automatically.

### Adding a Custom Tool

1. Add the tool definition to `TOOLS` array in `mcp-server/src/index.ts`
2. Add a handler function (e.g., `handleMyTool`)
3. Wire it in the `switch` statement in the tool dispatcher
4. Optionally add a one-liner to the thin prompt's tools table in `bootstrapService.ts`

## System Prompt vs Thin Prompt

| | Thin Prompt (CLAUDE.md) | System Prompt (prompt.md) |
|---|---|---|
| **Location** | Written to workspace root | `agent/system-prompts/prompt.md` |
| **When loaded** | Agent reads on every turn | Delivered once on first tool call |
| **Persistence** | On disk (auto-regenerated) | In memory only (never on disk) |
| **Content** | Tool routing, skill list, usage rules | Methodology, doctrine, behavioral guidelines |
| **Customization** | Edit `bootstrapService.ts` | Edit `prompt.md` |
